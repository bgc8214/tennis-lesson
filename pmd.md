# [제품 기획서] 오늘의 테니스 (Today's Tennis)
**AI 기반 테니스 레슨 복기 및 스윙 분석 웹 서비스 제품 기획서 (PMD)**

---

## 1. 개요 및 비전 (Executive Summary)

### 1.1 제품 개요
**'오늘의 테니스(Today's Tennis)'**는 테니스 플레이어가 비싼 비용을 지불하고 받는 1:1 레슨의 효과를 극대화하기 위한 **AI 기반 레슨 복기 및 스윙 데이터 분석 웹 SaaS 서비스**입니다. 사용자가 본인의 유튜브 레슨 영상 링크를 입력하거나 영상을 업로드하면, AI가 코치의 음성 피드백을 인식하여 정형화된 오답노트를 생성하고, 나아가 플레이어의 관절 궤적을 분석하여 과학적인 자세 교정 데이터를 제공합니다.

### 1.2 제품 비전
> **"레슨의 망각을 데이터의 자산으로"**

단순한 영상 저장소의 개념을 넘어, 플레이어의 성장 히스토리를 데이터로 추적하고 교정하는 **테니스 데이터 표준 플랫폼**으로 도약하는 것을 목표로 합니다.

## 2. 문제 정의 및 시장 기회 (Problem Statement & Market Opportunity)

### 2.1 기존 테니스 레슨 시장의 페인 포인트 (Pain Points)
1. **레슨 피드백의 높은 휘발성:** 테니스 레슨은 통상 시간당 8~12만 원 선의 고가 세션이지만, 코트 위에서 듣는 코치님의 피드백은 귀가하는 과정에서 90% 이상 망각됩니다.
2. **비정형 영상 데이터의 한계:** 수강생들은 복기를 위해 스마트폰으로 레슨 영상을 촬영하지만, 30분~1시간 분량의 긴 영상 속에서 "내가 어떤 지적을 받았고, 어떤 순간에 자세가 무너졌는지" 다시 찾아보기가 매우 고통스럽고 시간이 많이 소요됩니다.
3. **독학/개인 연습의 방향성 상실:** 레슨을 다녀온 후 개인 랠리나 볼머신 연습 시, 어떤 나쁜 습관을 집중적으로 교정해야 하는지 명확한 가이드라인(오답노트)이 없어 잘못된 폼을 반복 학습하게 됩니다.

### 2.2 시장 기회 (Market Opportunity)
* **테니스 동호인 인구의 급증:** 2030 세대를 중심으로 테니스 인구가 급증하고 있으며, 이들은 장비뿐만 아니라 실력 향상(실력 인바디)을 위한 테크 솔루션에 적극적으로 비용을 지불할 의사가 있습니다.
* **컴퓨터 비전 및 LLM 기술의 성숙:** 오픈소스 기반의 포즈 에스티메이션(Pose Estimation) 기술과 초거대 언어 모델(LLM)의 멀티모달 기능 고도화로, 과거 대형 연구소에서만 가능했던 관절 분석을 웹 환경에서 저비용으로 구현할 수 있는 기술적 적기를 맞이했습니다.

## 3. 핵심 가치 제안 (Core Value Propositions)

1. **시간 절약 및 복기 효율 극대화:** 1시간짜리 영상을 다 볼 필요 없이, AI가 요약한 3줄 오답노트와 타임스탬프 링크를 통해 1분 만에 레슨 복기가 가능합니다.
2. **시각 데이터 기반의 과학적 피드백:** 코치님이 말로 표현한 "타점이 낮다", "왼손이 일찍 떨어진다"는 지적을 실제 관절 각도 수치로 정량화하여 눈으로 확인시켜 줍니다.
3. **성장 궤적의 시각화:** 나의 고질병 지적 횟수 변화 추이와 관절 각도의 개선 흐름을 대시보드로 체계적으로 관리하여 실력 향상의 성취감을 부여합니다.

## 4. 제품 로드맵 (Product Roadmap)

제품의 시장 검증 속도와 기술적 복잡도, 인프라 비용 리스크를 고려하여 총 3단계로 빌드업합니다.

[Phase 1: MVP 단계] ➔ [Phase 2: Vision AI 단계] ➔ [Phase 3: 플랫폼 확장 단계]

### Phase 1: AI 레슨 요약 엔진 (MVP - 최소 기능 제품)
* **목표:** 복기 솔루션으로서의 시장성 및 유저 가치 검증 (나 자신의 도그푸딩 데이터 기준 개발).
* **핵심 기능:** 유튜브 링크 입력 ➔ 자막 및 오디오 스트림 파싱 ➔ AI 오답노트 리포트 발급.
* **비용 구조:** 유튜브 자막 API 및 구글 AI 스튜디오 무료 티어 활용으로 인프라 비용 **0원** 달성.

### Phase 2: 비전 AI 기반 관절 분석 (상용화 및 유료화 시작)
* **목표:** 시각 데이터 분석 도입을 통한 유료화 전환 및 기술 진입장벽 구축.
* **핵심 기능:** 영상 프레임 분석 ➔ 관절 포인트(21개 랜드마크) 추적 ➔ 스윙 메커니즘 수치화 및 텍스트 리포트 결합.
* **핵심 기술:** Google MediaPipe, YOLOv8 Pose.

### Phase 3: 무저장 직접 업로드 플랫폼 (스케일업 단계)
* **목표:** 유튜브 의존도를 탈피하고 독립적인 모바일 웹/앱 SaaS 서비스로 진화.
* **핵심 기능:** 클라이언트 웹 브라우저 단에서 오디오 선추출(FFmpeg.wasm) ➔ 서버 무저장(In-Memory) 처리 ➔ 누적 데이터 대시보드 구축 및 B2B 코칭 대시보드 판매.

## 5. 상세 기능 명세 (Feature Specifications)

### 5.1 [기능 1] 유튜브 링크 기반 레슨 파싱 & 요약 (Phase 1)
* **사용자 시나리오:** 유저가 유튜브에 올려둔 본인의 레슨 영상 주소(URL)를 복사하여 입력창에 넣고 '복기하기' 버튼을 누릅니다.
* **작동 프로세스:**
  1. 시스템은 `YouTube Transcript API`를 호출하여 동영상의 자동 생성 한글 자막 스크립트를 초고속 파싱합니다.
  2. 자막이 없는 영상일 경우, 오픈소스 `yt-dlp` 옵션을 활용하여 영상 전체가 아닌 오디오 스트림만 경량화하여 다운로드한 후, 내부 STT 모델을 거칩니다.
  3. 추출된 텍스트를 구글 Gemini LLM 프롬프트에 주입하여 정형화된 마크다운 템플릿 리포트로 반환합니다.

### 5.2 [기능 2] 컴퓨터 비전 기반 관절 궤적 및 스윙 분석 (Phase 2)
* **사용자 시나리오:** 오답노트에 연동된 주요 타임스탬프 구간을 클릭하면, 플레이어의 몸 위에 관절 링크 스켈레톤 라인이 오버레이된 분석 영상이 재생됩니다.
* **분석 매트릭스:**
  * **서브 타점 및 왼손 유지 각도:** 볼을 타격하는 순간 왼손이 하늘을 향해 유지되고 있는지 각도 측정.
  * **라켓 드롭(Racket Drop) 깊이:** 트로피 자세에서 라켓 헤드가 등 뒤로 얼마나 충분히 떨어져서 탄력을 받는지 깊이 분석.
  * **내전(Pronation) 여부:** 임팩트 직후 손목의 회전 각도를 프레임 단위로 판독.

### 5.3 [기능 3] 온디바이스 오디오 가공 및 무저장 파이프라인 (Phase 3)
* **용량 및 비용 제어 시스템:** 상용화 시 대용량 영상 업로드 비용을 방어하기 위한 필수 기술 명세입니다.
* **온디바이스 가공:** 사용자가 스마트폰 웹 브라우저에서 레슨 영상을 업로드할 때, `FFmpeg.wasm`을 이용하여 브라우저 자체적으로 1GB 영상을 10MB 오디오 파일로 0.5초 만에 변환 후 서버로 전송합니다.
* **In-Memory 처리:** 서버는 전송받은 오디오를 디스크에 영구 저장하지 않고 RAM 상에서 Whisper 및 Gemini API로 버스트 전송한 뒤 즉시 휘발시킵니다. 오직 '텍스트 레슨 노트' 데이터만 데이터베이스에 저장하여 영구 스토리지 보관 비용을 제로화합니다.

## 6. 기술 아키텍처 및 데이터 흐름 (Technical Architecture)

### 6.1 웹 서비스 기술 스택 (Tech Stack)

| 분류 | 기술 스택 (Tech Stack) | 주요 역할 |
| :--- | :--- | :--- |
| **Frontend** | Next.js, Tailwind CSS | 사용자 대시보드 및 반응형 모바일 UI 최적화 |
| **Backend** | Python FastAPI | AI 파이프라인 엔진 제어 및 백엔드 웹 API 구축 |
| **AI Engine** | Gemini 1.5 Pro/Flash, MediaPipe | 음성 맥락 요약 및 비전 기반 관절 데이터 추출 |
| **Database** | Supabase (PostgreSQL) | 사용자 정보, 결제 데이터, 레슨 텍스트 노트 저장 |
| **Storage** | Cloudflare R2 | 트래픽 비용(Egress Fee) 0원 기반 오디오/텍스트 보관소 |


## 7. 주요 화면 구조 기획 (UI/UX Wireframe)

### 7.1 메인 대시보드 화면 (Home)

* **상단:** 서비스 브랜드 로고 **[오늘의 테니스]**, 유저 프로필, 프리미엄 멤버십 구독 버튼.
* **중앙:** 거대한 텍스트 입력창과 카메라 아이콘. *"오늘의 레슨 복기를 시작하세요 (유튜브 링크 또는 영상 파일)"*
* **하단:** 최근 복기한 레슨 이력 카드 그리드 리스트 (예: `2026-06-03 서브 레슨 오답노트`, `2026-05-28 포핸드 타점 교정`).

### 7.2 레슨 상세 분석 리포트 화면 (Report View)

* **좌측 단 (영상 영역):** 관절 라인이 오버레이된 분석 비디오 플레이어. 타임스탬프 제어 인터페이스.
* **우측 단 (텍스트 노트 영역):** AI가 정제한 오늘의 3단 오답노트 카드.
* **Card 1 (고질병):** "토스 시 왼손이 너무 빨리 내려옵니다."
* **Card 2 (코치 큐잉):** *"라켓 헤드가 떨어질 때까지 왼손 끝으로 하늘을 찌르세요."*
* **Card 3 (액션 플랜):** "다음 개인 연습 시 첫 서브 20개는 무조건 왼손 고정에만 집중할 것."


* **최하단:** '카카오톡으로 공유하기', '텍스트 복사하기' 버튼 제공.

## 8. 비즈니스 및 수익 모델 (Monetization Model)

### 8.1 B2C 유저 구독 모델 (Freemium SaaS)

* **무료 등급 (Free):** 매월 레슨 복기 및 AI 텍스트 요약 노트를 기본 3회 제공.
* **베이직 패스 (월 4,900원):** 텍스트 오답노트 무제한 생성, 나의 고질병 키워드 분석 누적 대시보드 이용 가능.
* **프로 AI 패스 (월 9,900원):** 프로 에디션 기능인 **Vision AI 기반 관절 궤적 및 스윙 각도 정밀 리포트** 제공.

### 8.2 B2B 코칭 연동 비즈니스 (Future Expansion)

* **테니스 코치 전용 대시보드 판매:** 테니스 아카데미 및 코치들이 수강생들의 진도와 피드백 영상을 체계적으로 관리할 수 있는 CRM 툴로 확장합니다. 코치가 수강생에게 "내가 말로 백번 설명하는 것보다 이 앱 화면으로 관절 각도 보면서 연습하세요"라며 인바디처럼 영업을 유도하는 생태계를 구축합니다.

## 9. 핵심 성과 지표 (KPIs)

1. **제품 유지율 (Retention Rate):** 유저가 다음 주 레슨 후 다시 서비스를 이용하는 비율 (목표: 주간 리텐션 60% 이상).
2. **AI 리포트 만족도 (Clarity Score):** 사용자가 발급된 오답노트에 대해 피드백하는 '도움이 되었음(Thumbs Up)' 비율.
3. **분석 파이프라인 비용 (COGS):** 영상 처리 1회당 서버 및 API 비용 지출 규모 (목표: 영상 1회당 30원 이하 원가 방어).
