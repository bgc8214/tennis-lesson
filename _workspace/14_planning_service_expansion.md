# 14. 서비스 확장 기능 기획 — 레슨 복기를 넘어선 "테니스 성장 플랫폼"

작성: 오케스트레이터 세션 / 2026-07-17
성격: **기획 (미구현)** — 09문서 2절(리포트 주변 기능), 13문서(전사 한계 대응)와
겹치지 않는, 서비스 전체 관점의 확장 기능 탐색
평가 기준: **데이터 해자**(축적된 내 레슨 데이터를 쓰는가) / 비전 정합(망각→자산) /
리텐션 / 수익 연결 / 난이도

## 0. 관점 — 우리의 유일한 자산은 "축적된 나의 레슨 데이터"다

경쟁자가 못 베끼는 것은 파이프라인이 아니라 **유저별로 쌓인 지적/교정/성장
기록**이다. 확장 기능의 제1 기준은 "이 데이터를 쓰는가". 데이터를 안 쓰는
기능(코트 예약, 파트너 매칭 등)은 레드오션 유틸리티라 원칙적으로 하지 않는다(6절).

유저의 테니스 생활 전체를 저니로 놓고 본다:

```
[레슨 전] → [레슨] → [복기] → [개인 연습] → [게임/시합] → [장비/몸 관리] → (반복, 성장)
   현재 서비스는 "복기" 한 칸만 점유 — 나머지 칸에 데이터 해자를 확장할 기회
```

## 1. Tier 1 — 데이터 해자 직결, 우선 검토 (유료화 견인)

### 1-1. AI 코치 챗 — "내 레슨을 전부 기억하는 조수" ★ 최우선 후보

- **무엇**: 축적된 내 리포트/타임스탬프/전사 위에서 대화형 Q&A.
  "지난달에 코치님이 서브 토스 관련해서 뭐라고 했지?", "요즘 발리에서 자주
  지적받는 게 뭐야?", "다음 개인 연습 뭐 하면 좋을까?"
- **왜 강력한가**: 데이터가 쌓일수록 답이 좋아짐 = 이탈 비용 증가 = 구독 유지
  장치. PMD 비전("망각을 자산으로")의 최종형. 기술적으로도 기존 자산(리포트
  DB + Gemini)만으로 구현 가능 — 신규 파이프라인 불필요.
- **할루시네이션 원칙 계승**: 답변은 반드시 출처 리포트/타임스탬프 인용과 함께
  (1-6 2계층 원칙의 챗 버전). 근거 없는 일반 조언은 "AI 일반 지식" 라벨.
- 난이도: 중 / 수익: 베이직 이상 전용 기능으로 적합

### 1-2. 맞춤 유튜브 레슨 큐레이션 — 고질병 기반 추천

- **무엇**: 내 반복 지적 키워드("왼손 하강", "라켓드롭")로 공개 유튜브
  레슨 영상을 큐레이션해 리포트 하단/주간 알림에 노출.
  "이번 주 고질병 보완 영상 3개".
- **왜**: 구현이 매우 쌈(키워드 → YouTube 검색 API)에 비해 체감 가치 큼.
  "복기 앱"에서 "성장 코치"로 포지셔닝 확장하는 가장 싼 다리.
- 난이도: 하 / 리스크: 추천 품질 관리(테니스 채널 화이트리스트로 시작)

### 1-3. 연말/월간 결산 리포트 — "테니스 Wrapped"

- **무엇**: 월/연 단위 자동 결산 — 총 레슨 수, 최다 지적→졸업한 고질병,
  스윙 수(13문서 축2 연동), 성장 곡선. 카드 뉴스형 이미지로 생성.
- **왜**: Spotify Wrapped가 증명한 공유 바이럴 포맷 + 12문서 마케팅
  루프(카톡 공유)와 직결. 연말 시즌 신규 유입 이벤트로 활용.
- 난이도: 하~중 (데이터는 이미 있음, 이미지 템플릿 작업이 대부분)

### 1-4. 개인 연습 세션 기록 — 저니의 "연습" 칸 점유

- **무엇**: 레슨이 아닌 개인 연습(볼머신, 벽치기, 랠리)도 기록 단위로 수용.
  영상 있으면 스윙 통계(13문서 축2 재사용), 없으면 연습 체크리스트(09문서 #2)
  수행 기록 + 셀프 메모(13문서 축3)만으로도 성립.
- **왜**: 레슨은 주 1회지만 연습은 주 2~3회 — **주간 접점이 2~3배**로 늘어나는
  리텐션 기능. "레슨 복기 앱"→"테니스 기록 앱"으로 TAM 확장.
- 난이도: 중 (기존 컴포넌트 재조합)

## 2. Tier 2 — 성장 가시화 (리텐션 보강)

### 2-1. 레벨/NTRP 자가 진단 트래킹

- 분기별 자가 진단 설문 + 지적 감소 추이를 결합한 "실력 인바디 점수".
  객관 지표 아님을 명시(자가 진단 기반)하되, 그래프가 우상향하는 경험 제공.
- 난이도: 하 / 주의: 점수 산정 로직이 엉터리면 신뢰 훼손 — 단순·투명하게

### 2-2. 뱃지/스트릭 게이미피케이션

- 복기 스트릭(N주 연속), 고질병 졸업 뱃지, 연습 체크리스트 완주 뱃지.
  09문서 #1(고질병 트래커)의 보상 레이어.
- 난이도: 하 / 과하면 유치해짐 — 미니멀하게

### 2-3. 시합/게임 기록 (동호회 유저)

- 동호회 시합 스코어 + (선택) 영상 링크 기록. 게임레슨 리포트와 연결해
  "연습에서 지적받은 것이 시합에서 나왔는가" 회고.
- 난이도: 중 / Tier 1 검증 후 — 시합 유저 비중을 베타에서 확인하고 결정

## 3. Tier 3 — 부가 유틸리티 (가볍게, 방어적으로만)

- **장비 기록**: 라켓/스트링/텐션 + 교체 주기 알림. 데이터 해자와 무관하지만
  구현 반나절 수준이라 "채워두는" 가치. (난이도: 최하)
- **부상 예방 정보**: 지적 패턴(손목 과사용 등)과 연계한 스트레칭/예방 콘텐츠.
  **의료 조언 아님 고지 필수**, 진단형 표현 금지 — 정보 제공에 한정. (난이도: 하,
  리스크 관리 중요)
- **용어 카드**(09문서 #8)는 이 티어와 결합해 "테니스 지식 코너"로 묶을 수 있음.

## 4. Tier 4 — 플랫폼 단계 (Stage 4 이후에만)

- **코치 마켓플레이스**: 리포트 데이터 기반 "이 고질병 잘 고치는 코치" 매칭.
  B2B(12문서 Stage 4) 코치 풀이 생긴 뒤에만 성립 — 지금은 기획 금지.
- **레슨 노트 공개 피드**: 유저가 익명으로 오답노트를 공개/탐색 ("남들은 뭘
  지적받나"). 커뮤니티 콜드스타트 리스크가 커서 유저 수 확보 후.

## 5. 우선순위 종합 (기존 백로그와 통합)

| 순위 | 기능 | 출처 | 근거 |
|---|---|---|---|
| 1 | 연습 체크리스트 + 클립 공유 | 09문서 #2/#4 | 기결정 (12문서 Stage 0) |
| 2 | 코칭 모먼트 내비게이션 | 13문서 축1 | 품질 한계 대응, 거의 공짜 |
| 3 | **AI 코치 챗** | 본 문서 1-1 | 데이터 해자 최종형, 유료화 견인 |
| 4 | 유튜브 큐레이션 | 본 문서 1-2 | 최저 비용 대비 포지셔닝 확장 |
| 5 | 개인 연습 세션 기록 | 본 문서 1-4 | 주간 접점 2~3배 |
| 6 | 테니스 Wrapped | 본 문서 1-3 | 바이럴 이벤트 (연말 타이밍 맞춰) |
| 7 | 고질병 트래커 | 09문서 #1 | 유료화 시점 (기결정) |
| 8 | 레벨 트래킹/뱃지 | 본 문서 2-1/2-2 | 리텐션 보강 |

## 6. 하지 않을 것 (명시적 제외)

| 기능 | 제외 이유 |
|---|---|
| 코트 예약 | 레드오션(기존 앱 다수), 데이터 해자 무관, 운영 부담(코트 DB 관리) |
| 파트너/게임 매칭 | 커뮤니티 콜드스타트 실패 확률 높음, 비전 이탈 |
| 실시간 레슨 스트리밍/중계 | 인프라 비용 폭발, 무저장 원칙과 충돌 |
| 프로 경기 분석 콘텐츠 | 저작권 리스크 + 우리 데이터 해자 무관 |
| 자체 커머스(장비 판매) | 완전 별개 사업 — 제휴 링크 수준까지만 |

"하지 않을 것"을 문서화하는 이유: 1인 개발에서 가장 비싼 실수는 잘못된 기능을
잘 만드는 것. 요청/유혹이 생길 때 이 표를 먼저 볼 것.
