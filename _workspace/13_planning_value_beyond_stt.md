# 13. 전사 품질 한계를 감안한 다중 가치 축 기획 — Graceful Degradation 전략

작성: 오케스트레이터 세션 / 2026-07-17
성격: **기획 (미구현)** — 배경은 11문서(인사이트 ②), 실측 근거는 골든셋/ac833ac 커밋
상태: 구현 위임 전. 각 축의 섹션 번호로 위임 지시

## 0. 배경 — 왜 이 문서가 필요한가

실측으로 확인된 사실:

- 코트 원거리 마이크 오디오는 물리적 한계가 있다 (speech_prob 0.076,
  mean -27~-40dB — VAD 감지 범위 밖, 42b3a8a 커밋 근거)
- 소프트웨어 처방 중 전처리·용어사전은 실측 반증됨 (ac833ac), 남은 카드는
  모델 상향(large-v3, 미실측)이지만 그것도 잔향/원거리 녹음을 완전히 이길 수 없음

**전략 전환**: `전사 품질 = 서비스 품질`인 단일 기둥 구조를 버리고,
오디오 품질이 나빠도 살아남는 **다중 가치 축 + 우아한 품질 저하(graceful
degradation)** 구조로 재편한다. 어떤 영상이 들어와도 빈손으로 돌려보내지 않는다.

## 1. 축 1 — 코칭 모먼트 내비게이션 ("들어주는 AI" → "들을 곳을 찾아주는 AI")

**우선순위 1위. 표시 전략 전환이라 비용이 가장 쌈.**

- 근거: 전사 내용이 깨져도 "코치가 말한 시점"의 탐지는 견고하다
  (발화 구간 탐지 ≫ 내용 인식 난이도). 유저는 자기 코치 목소리를 알아듣는다 —
  문제는 1시간 중 "어디서 말했는지 찾기"였고, 그것만 AI가 해결해도 핵심
  페인포인트(복기 시간)는 풀린다.
- 기획:
  - 전사 신뢰도가 낮은 항목은 깨진 인용을 숨기고 "🔊 코칭 구간 (20:13)" +
    러프한 카테고리 라벨만 노출 → 할루시네이션 노출 리스크 구조적으로 제거
  - 리포트 모드 자동 전환 + 정직한 안내 문구: 신뢰도 높음 → 풀 오답노트(현행),
    낮음 → "이번 영상은 음성이 멀어서 코칭 구간 안내 중심으로 보여드려요"
  - 판정 신호: 세그먼트 avg_logprob 평균, 검증 통과율, match_score 분포 등
    이미 수집 중인 지표 재사용 (신규 계측 불필요)
- 구현 위치: 백엔드는 리포트 메타에 `transcript_quality` 등급 추가,
  프론트 ReportView 모드 분기. backend+frontend 병행.

## 2. 축 2 — 시각 가치: 영상은 오디오와 달리 품질이 멀쩡하다

원거리 촬영에서 목소리는 죽지만 몸은 화면에 잘 나온다. 오디오 무관 가치이며,
오디오가 나쁜 영상일수록 Phase 2(비전)를 앞당길 이유가 된다.

우선순위 순:

1. **스윙 통계 — "테니스 인바디"** (Phase 2 축소판, 관절 분석보다 훨씬 쉬움):
   오늘 스윙 횟수, 포핸드/백핸드 비율, 랠리 길이, 실제 운동 시간 vs 휴식.
   코칭 발언이 하나도 안 잡혀도 "오늘 레슨 정량 리포트"가 성립.
   PMD의 "실력 인바디" 포지셔닝 그 자체.
2. **자동 하이라이트 컷**: 스윙/랠리 구간 시각 감지 → "오늘의 내 포핸드 모음"
   자동 클립. 공유 가치(카톡) 높음 — 12문서 마케팅 루프와 결합.
3. **주간 폼 비교**: 같은 스트로크의 지난주 vs 이번주 나란히 재생.
4. 관절 각도 정밀 분석(Phase 2 본편, 프로 패스 유료 기능)은 그 다음.

- 담당: vision 에이전트 (mediapipe-pose 스킬). 1번은 포즈 추정 없이
  모션/공 소리(오디오 이벤트 감지는 발화 인식과 달리 저품질 오디오에서도
  견고)만으로도 가능한지 선행 검토.

## 3. 축 3 — 셀프 음성 메모: **강등 (2026-07-19 재판정)**

> **[2026-07-19 재판정] "상" 승격 철회 → 강등.** 기술적으로는 유일한 깨끗한
> 오디오 소스라 매력적이지만, 행동 관점 스트레스 테스트 결과 주력 기능으로는
> 부적합하다고 판단해 아래로 대체한다.

**철회 근거:**

1. **서비스 존재 이유와 정면 모순.** PMD 출발점은 "복기는 귀찮아서 안 한다"인데,
   셀프 메모는 유저에게 망각 전에 능동적 복기를 요구한다 — 제품이 자동화하겠다고
   약속한 노동을 유저에게 되돌리는 구조. "링크만 넣으면 끝"이라는 핵심 UX
   약속과 충돌.
2. **쓸 사람은 애초에 필요 없는 사람.** 레슨 직후 꼬박꼬박 메모를 남길 성실한
   유저는 이미 수첩에 적던 사람들 — 이 앱이 가장 덜 필요한 세그먼트다. 진짜
   타겟("다 까먹는 대다수")은 메모도 안 남긴다.
3. **타이밍 최악.** 레슨 직후는 땀 닦고 교대하고 이동하는 시간이고, 코트 앞
   혼잣말은 정서적으로 민망하다. 그 순간을 놓치면 가치가 사라지는데,
   "지금 메모하세요" 알림은 잔소리로 소비된다.
4. **노동-보상 비대칭.** 유저가 코치 말을 떠올려 말할 수 있다면 이미 머릿속
   복기는 끝난 것 — 메모는 유저보다 앱(데이터)에게 이득인 구조. 즉각적이고
   눈에 보이는 보상 없는 노동 요구 기능은 저널링 앱들이 반복 증명했듯 죽는다.

**대체 카드 (행동 프로필이 더 우월한 순):**

- **근접 마이크 촬영 (2-D 촬영 가이드, 이미 구현됨)** — 매 레슨 노력이
  필요한 셀프 메모와 달리, 폰 위치는 한 번 습관 들이면 이후 영원히 수동적.
  일회성 셋업 > 반복 노동. 정확한 내용 공백을 메우는 진짜 해법.
- **👍/👎 탭 (신규, 최저 마찰)** — 유저 입력 중 마찰이 가장 낮은 형태.
  카드별로 "이 피드백 도움됐어요?" 탭 하나. 셀프 메모보다 먼저 시도.
- **텍스트 한 줄 수요 테스트 (강등된 셀프 메모의 저비용 검증)** — 녹음 UI를
  만들기 전에 "오늘 기억나는 지적이 있나요? (선택)" 텍스트 입력 한 줄로
  수요를 먼저 잰다. 텍스트도 안 쓰는 유저는 음성도 안 한다 — 사실상
  무비용 실험. 입력률이 유의미하면 그때 음성으로 재승격 검토, 아니면
  완전 폐기.
- **코치 한 줄 코멘트(축4)** — 코치는 B2B 맥락에서 쓸 유인이 있어 행동
  문제가 다르다. 셀프 메모 대신 이쪽에 자원 배분.

## 4. 축 4 — 코치 입력: 정답을 아는 사람이 이미 있다 (09문서 #5 확장)

- 코치 확인 링크에 "한 줄 코멘트" 입력 필드만 추가 → 코치가 쓴 텍스트는
  검증이 필요 없는 골든 데이터이자 리포트 최상단 가치.
- B2B(12문서 Stage 4) 접점과 동일 기능이므로 중복 투자 없음.

## 5. 통합 — 품질 저하 스택

| 오디오 상태 | 제공 가치 |
|---|---|
| 좋음 | 풀 오답노트 (현행 whisper 검증 경로) |
| 나쁨 | 코칭 모먼트 내비게이션(축1) + 스윙 통계/하이라이트(축2) + 👍/👎 + (수요 확인되면) 텍스트 메모 |
| 최악(발화 추출 0) | 스윙 통계 + 하이라이트 + 👍/👎 |

+ **촬영 가이드** (온보딩 1장: "폰을 코치 쪽 펜스에", "가능하면 네트 근처에")
— 입력 자체를 개선하는 가장 싼 품질 수단. 가이드를 따른 유저는 자연히
상위 티어 경험 → 가이드 준수 유인이 됨.
