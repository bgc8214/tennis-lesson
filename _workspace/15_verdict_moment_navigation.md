## 1. 판정

### 1-1. quote 정밀도 13~14%는 "실패"가 아니라 "제품 형태 오류"라는 신호다

검토된 hallucination 항목들의 `correct_quote`를 나란히 보면 패턴이 명확하다:

| 파이프라인 인용 (오류) | 실제 발언 (사람 검토) | sec |
|---|---|---|
| "바퀴 전체가 소음을 생겨놓고" | "라켓 전체가 손을 세워놓고" | 82 |
| "삼각형 키고" | "팔 더 피고" | 1157 |
| "누가 밟아야 되는거지" | "누가 발리를 그렇게 하냐" | 1054 |
| "제가 공주면 앞으로 완전히 들어와서 내리고" | "제가 공주면 앞으로 원투 들어와서 때리고" | 312 |

**텍스트(낱말)는 대부분 틀렸지만, 타이밍은 사실상 전부 맞았다.**
검토 22건 중 거의 전부가 "실제 코치 피드백이 있었던 순간"을 정확히 가리키고
있고(두 영상 합쳐 놓친 피드백 기록은 1건), 실패는 받아쓰기에만 있다.

### 1-2. 결론 (3줄)

1. **"코치 말씀 인용" 제품은 이 녹음 환경에서 종료** — 정밀도 13%짜리 문장을
   "코치가 이렇게 말했다"로 노출하는 것은 신뢰 자살. STT 추가 실험(2절)과
   무관하게 인용 원문 노출은 기본 OFF가 되어야 한다.
2. **"코칭 모먼트 내비게이션"(13문서 축1)은 가설이 아니라 검증된 사실** —
   모먼트 정밀도 기준 ~95%로 이미 합격. 지금 파이프라인 그대로 표시 전략만
   바꾸면 오늘 당장 정직한 제품이 된다.
3. 검증 게이트의 보증 범위(전사 충실성 ≠ 현실 충실성, 11문서 인사이트 ②)가
   실측으로 확정됐다 — match_score는 "AI 왜곡"만 잡고 "STT 오인식"은 못 잡는다.

## 2. 실행 항목 (로컬 세션 착수용)

### 2-A. [주 노선, 최우선] 코칭 모먼트 내비게이션 구현 — 13문서 축1

**백엔드** (`gemini_service.py` whisper 경로 + `lessons.py`):

1. 리포트에 전사 품질 등급 산출·저장: 신호는 이미 수집 중인
   `avg_logprob` 평균 / 검증 통과율 / match_score 분포 재사용.
   제안: `transcript_quality: "high" | "low"` (임계값은 골든셋 2개 영상이
   low로 판정되도록 보정 — 이 둘이 low의 기준 표본).
2. `stt_stats`/`verification` 통계의 DB 영속화 (`lesson_reports`에 jsonb 컬럼
   추가, migration 파일 `_workspace/15_quality_meta_migration.sql`) —
   품질 등급 판정과 향후 대시보드의 공통 기반.
3. timestamps 항목에 인용 노출 여부와 무관한 표시용 필드는 추가하지 않는다 —
   판단은 프론트가 `transcript_quality`와 항목별 `match_score`로 한다
   (API shape 최소 변경 원칙).

**프론트** (`FeedbackTimeline.tsx`, `NoteCards.tsx`):

4. `transcript_quality=low`이면:
   - quote 원문 숨김 → "🔊 코칭 구간 20:13 — 발리 자세 관련" 형태
     (카테고리/label만, **"AI 추정" 라벨 필수**) + 유튜브 타임스탬프 점프
   - 상단 안내: "이 영상은 음성이 멀어 코칭 구간 안내 중심으로 보여드려요"
   - 카드(card1~3)는 evidence 인용 대신 취지 요약 + "구간 듣기" 링크
5. `transcript_quality=high`면 현행 풀 오답노트 유지.
6. 원칙: 낱말이 틀려도 취지/카테고리는 대체로 유효("팔 펴서…"→발리 교정)
   — 러프 라벨로 쓰되 코치 발언인 척하지 않는 것이 전부다.

### 2-B. [측정 체계] `moment_valid` 라벨 도입

- 골든셋 스키마에 `moment_valid: true|false` 추가 — "이 sec에 실제 코치
  피드백이 있었는가" (quote 정오와 독립).
- 기존 검토분은 소급 가능: `correct_quote`가 채워진 hallucination = moment_valid
  true로 초기화 후 확인.
- `eval_golden.py`에 **moment precision** 지표 추가 — 내비게이션 제품의 KPI는
  quote precision이 아니라 이것. 현재 데이터 기준 ~95% 예상.
- 남은 골든셋 2개(`wh2b6vyr_ys`, `aya3iilw2b0`) 검토 시 이 라벨까지 채우기.

### 2-C. [STT 탐색 종료 실험 — 딱 2개만, 기대치 낮게] ✅ 종료(1건 실행, 결론 도출)

1. **Groq `whisper-large-v3-turbo`** ✅ 실행 완료, **반증**. 골든셋 2개 영상
   (`4mr0tVIu9sw`, `5PUGx-OYI5s`)을 `STT_PROVIDER=groq`로 재실행해 correct_quote와
   대조 — whisper medium보다 낫지 않고 recall이 오히려 악화(다수 검증 지점에서
   세그먼트 자체가 드롭). large-v3 계열 반복 루프(09문서 1-8)도 turbo에서 재현.
2. **Gemini 오디오 청크 경로 재채점**: Groq 반증으로 STT 모델 탐색 자체를
   종료하기로 판단 — **실행하지 않음**. 2-A가 이미 기본 제품으로 성립해
   블로킹 요소가 아니므로, 추가 STT 모델 쇼핑보다 다른 우선순위에 자원을
   쓰는 쪽을 택함.

### 2-D. [입력 개선] 촬영 가이드 (13문서 5절 승격)

- 사람 귀는 알아들었다 = 오디오가 한계선 *근처*라는 뜻. 마이크가 조금만
  가까워져도 STT 티어가 바뀔 가능성이 크다.
- 온보딩/업로드 화면에 1장: "폰을 코치 쪽 펜스에", "가능하면 네트 근처",
  "바람 심한 날은 마이크 방향 주의". 가이드 준수 영상이 들어오면
  `transcript_quality=high` 경험 → 준수 유인 자연 형성.

## 3. 하지 않을 것 (이 판정으로 종료되는 것들)

| 항목 | 이유 |
|---|---|
| 인용 원문 기본 노출 | 정밀도 13~14% 실측 — 신뢰 파괴 |
| STT 모델 추가 상향/교체 (large-v3, Groq turbo 등) | 반복 루프 반증(09문서 1-8) + Groq도 recall 악화로 반증(15문서 2-C). 추가 모델 쇼핑 금지 — 새 가설 없이는 재실험 안 함 |

| LLM으로 깨진 인용 "매끄럽게 교정" | 할루시네이션 뒷문. 교정 추정은 "AI 추정" 라벨 하에 label/취지 요약으로만 |
